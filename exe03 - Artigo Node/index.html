Artigo Node.JS

O Node.js é um open-source, cross-platform, back-end, JavaScript. Seu ambiente de execução que é executado no interpretador V8 (é o interpretador JavaScript, também chamado de máquina virtual Javascript (ou engine), desenvolvido pela Google e utilizado em seu navegador Google Chrome.) e executa o código JavaScript fora de um navegador web .


O Node.js foi escrito inicialmente por Ryan Dahl em 2009, cerca de treze anos após a introdução do primeiro ambiente JavaScript do lado do servidor, o LiveWire Pro Web da Netscape. O lançamento inicial suportava apenas Linux e Mac OS X. Seu desenvolvimento e manutenção foram liderados por Dahl e mais tarde patrocinados por Joyent.

Dahl criticou as possibilidades limitadas do servidor web mais popular em 2009, Apache HTTP Server , para lidar com muitas conexões simultâneas até 10.000 ou mais, e a forma mais comum de criar código de programação sequencial, quando o código bloqueia o processo ou várias pilhas de execução implícitas no caso de conexões simultâneas.

Dahl demonstrou o projeto no inaugural JSConf Europeu, em 8 de novembro de 2009. Node.js combinado com Google V8 interpretador de JavaScript.

Em janeiro de 2010, um gerenciador de pacotes foi introduzido para o ambiente Node.js chamado npm. O gerenciador de pacotes torna mais fácil para os programadores publicar e compartilhar o código-fonte de pacotes Node.js e é projetado para simplificar a instalação, atualização e desinstalação de pacotes.

Em junho de 2011, a Microsoft e Joyent implementaram uma versão nativa do Windows do Node.js. A primeira versão do Node.js com suporte ao Windows foi lançada em julho de 2011.

Em janeiro de 2012, Dahl se afastou, promovendo o colega de trabalho e criador do npm Isaac Schlueter para gerenciar o projeto. Em janeiro de 2014, Schlueter anunciou que Timothy J. Fontaine lideraria o projeto.

Em dezembro de 2014, Fedor Indutny iniciou o io.js, um fork do Node.js. Devido ao conflito interno sobre a governança de Joyent, io.js foi criado como uma alternativa de governança aberta com um comitê técnico separado. Ao contrário do Node.js, os autores planejaram manter o io.js atualizado com os últimos lançamentos do interpretador JavaScript V8 do Google.

Em fevereiro de 2015, a intenção de formar uma Fundação Node.js neutra foi anunciada. Em junho de 2015, as comunidades Node.js e io.js votaram para trabalhar juntas sob a Fundação Node.js.

Em setembro de 2015, Node.js v0.12 e io.js v3.3 foram mesclados novamente no Node v4.0. Essa fusão trouxe recursos do V8 ES6 para o Node.js e um ciclo de lançamento de suporte de longo prazo. A partir de 2016, o site io.js recomenda que os desenvolvedores voltem para o Node.js e que nenhuma outra versão do io.js seja planejada devido à fusão.

Em fevereiro de 2016, Node.js 0.10.42 é lançado. Em 2019, a JS Foundation e a Node.js Foundation se fundiram para formar a OpenJS Foundation .
